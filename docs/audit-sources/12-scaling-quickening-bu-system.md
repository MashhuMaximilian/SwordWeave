## 1. Purpose

This page defines how **capabilities scale in strength, speed, and execution intensity** within the BU system.

It governs:

- Upcasting / strengthening effects
- Speed of execution changes (quickening)
- Intensity modifiers
- BU-based scaling logic

---

## 2. CORE PRINCIPLE

```
Scaling does not change the capability.
It modifies how the same capability is expressed.
```

---

## 3. SCALING TYPES

### 🔺 3.1 Intensity Scaling

Increases raw power of effect.

Examples:

- more damage dice
- stronger conditions
- larger AoE impact

Rule:

- costs additional BU proportional to upgrade tier

### ⏱ 3.2 Quickening (Speed Scaling)

Modifies execution speed.

```
Quickening = reducing casting time OR reaction delay
```

Stages:

- Standard
- Fast
- Instant
- Reaction-level

Rule:

- faster execution = higher BU cost

### 🌍 3.3 Range Scaling

Modifies spatial reach.

- Touch
- Close
- Near
- Far
- Very Far
- Extreme

Rule:

- each tier increase adds BU cost

---

## 4. BU SCALING RULE

```
Scaling Cost = Base Capability BU × Scaling Modifier
```

Modifiers:

- Intensity +X BU per tier
- Range +X BU per tier
- Speed +X BU per tier

---

## 5. STACKING RULE

Scaling effects stack unless explicitly restricted.

Example:

- Fast + High Intensity is allowed
- Fast + Fast upgrades stack into Instant (if system allows tier jump)

---

## 6. QUICKENING SYSTEM (SPECIAL RULE)

Quickening follows strict escalation path:

```
Standard → Fast → Instant → Reaction
```

Each step:

- increases BU cost exponentially or tier-based

---

## 7. CONCENTRATION / SUSTAINED SCALING

Sustained effects may require:

- upkeep cost (BU per turn or scene)

If no upkeep:

- strain increases instead (DM layer)

---

## 8. COMBAT INTERACTION RULE

Scaling affects:

- damage output
- condition strength
- AoE coverage
- timing advantage

BUT does NOT change:

- domain identity
- source type

---

## 9. BU + QUICKENING RELATION

```
Faster = more BU cost
Stronger = more BU cost
Longer = optional upkeep or strain
```

---

## 10. DESIGN PRINCIPLE

```
Scaling is a multiplier of expression, not a rewrite of capability.
```
