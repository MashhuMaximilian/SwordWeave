## 🧠 Base Principle

```
DC = 5 + Attribute Base + Proficiency + Modifiers
```

---

## 📊 ATTRIBUTE BASE (Resolution Anchor)

| Attribute Type | Meaning |
| --- | --- |
| Physical | Strength / agility / bodily execution |
| Magical | Structured energy / domain manipulation |
| Abstract | Mind / will / conceptual influence |

---

## ⚙️ DC BASE VALUES

| Difficulty Level | Base DC |
| --- | --- |
| Trivial | 5 |
| Easy | 10 |
| Standard | 15 |
| Hard | 20 |
| Extreme | 25 |
| Mythic | 30+ |

---

## 🧩 MODIFIER LAYERS

| Modifier Type | Effect on DC |
| --- | --- |
| Environmental Pressure | +1 to +5 |
| Target Resistance | +2 to +10 |
| Capability Complexity | +2 to +8 |
| Opposing Capability | replaces base DC entirely (contest mode) |
| Advantage State | -2 to -5 effective DC |
| Disadvantage State | +2 to +5 effective DC |

---

## 🔺 SCALING INTERACTION RULE

- Speed scaling does NOT affect DC
- Range scaling does NOT affect DC
- Intensity scaling MAY affect DC if it changes opposition difficulty (DM adjudicated)

# 🛡 SAVE SYSTEM TABLE

## 🧠 Core Idea

Saves are **reactive DC checks against incoming capabilities**

---

## 🎲 SAVE TYPES

| Save Type | Attribute Used | Meaning |
| --- | --- | --- |
| Physical Save | Physical | Resist force, movement, bodily effects |
| Magical Save | Magical | Resist structured energy / spells |
| Mental Save | Abstract | Resist influence, fear, control |

---

## ⚖️ SAVE FORMULA

```
d20 + Attribute + Proficiency (if trained) + Modifiers ≥ DC
```

---

## 🧩 SAVE OUTCOMES

| Result | Outcome |
| --- | --- |
| Success | Full or partial resistance |
| Failure | Full effect applies |
| Critical Success | Negate + gain resistance buffer |
| Critical Failure | Enhanced effect or extended duration |

---

## 🔥 SAVE SCALING RULE

- Higher BU capabilities increase DC
- High complexity adds passive DC bonus
- Certain tags bypass saves entirely (defined later)

# ⚔️ CONTEST RESOLUTION TABLE

## 🧠 Core Idea

Used when two capabilities directly oppose each other.

---

## ⚔️ RESOLUTION FORMATS

### Mode 1 — DC Contest

```
Attacker DC vs Defender Roll
```

- Used when one side initiates effect

### Mode 2 — Opposed Rolls

```
d20 + mods vs d20 + mods
```

- Used for simultaneous capabilities

### Mode 3 — Capability Override

```
Higher tier capability overwrites lower tier automatically
```

- Used for extreme BU disparities

---

## 🧩 CONTEST MODIFIERS

| Factor | Effect |
| --- | --- |
| Higher BU | + advantage to roll or +DC |
| Better Scaling | +2 to +6 effective edge |
| Reaction Timing | interrupts opponent advantage |
| Resistance Type Match | reduces effectiveness |

---

## ⚡ CRITICAL RULE

- Contest is NOT damage comparison
- It is **intent resolution priority system**

# 📊 MODIFIER SYSTEM TABLE

## 🧠 Core Idea

Modifiers are layered bonuses applied to rolls or DC.

---

## 🧩 MODIFIER TYPES

| Type | Source | Effect |
| --- | --- | --- |
| Attribute Modifier | character stats | +1 to +5 typical |
| Capability Modifier | BU purchases | +1 to +10 |
| Item Modifier | equipment | +1 to +5 |
| Environmental Modifier | scene conditions | -5 to +5 |

---

## ⚖️ STACKING RULE

- All modifiers stack additively
- Exception: identical source type does not stack unless specified

---

## 🔥 DESIGN RULE

Modifiers NEVER change capability identity — only resolution probability.

# 🎲 ADVANTAGE / DISADVANTAGE TABLE

## 🧠 Core Idea

A simplified probability shift system replacing rerolls or complexity.

---

## 📊 EFFECTS

| State | Effect |
| --- | --- |
| Advantage | +3 effective roll bonus OR roll twice take higher |
| Disadvantage | -3 effective roll penalty OR roll twice take lower |

---

## ⚖️ APPLICATION RULE

- Cannot stack beyond one advantage/disadvantage state
- Cancels out if both present
- Applies after modifiers

---

## 🧩 DESIGN NOTE

This system is intentionally lightweight:

- avoids reroll math bloat
- preserves BU system dominance
