# 🎮 CHARACTER CREATION FLOW (v1)

This page defines the **player-facing character creation system** built on the BU + Grammar framework.

---

# 🧠 DESIGN GOAL

Players can create a functional character in ~10–20 minutes using either:

- Library heritage (fast mode)
- Full custom grammar construction (advanced mode)

The system remains modular, intuitive, and BU-driven.

---

# ⚙️ STEP 0 — CREATION MODE

## A) Library Mode

- Choose race template
- Choose background template
- Optionally choose archetype/class template
- All auto-fills base structure

## B) Custom Mode

- Fully build race/background/class equivalents
- Uses DBP system + DM validation

---

# 🧬 STEP 1 — CORE CONCEPT

Player defines:

- Character idea (what they are)
- Core fantasy (what they do)
- Style (how they do it)

No mechanics yet.

---

# ⚔️ STEP 2 — BASE FRAME (DBP)

If using heritage:

- Race-like traits
- Background traits
- Archetype modifiers

If custom:

- Constructed via DBP rules

Provides:

- narrative identity
- minor mechanical traits
- starting proficiencies

---

## Step 3 — Assign Attributes & Proficiency

Distribute **+10 total points** between the three core attributes, then distribute each attribute's modifier across its sub-Practices.

- 🟠 **Physical** (-1 to +5)
- 🔵 **Mental** (-1 to +5)
- 🟣 **Magical/Abstract** (-1 to +5)

Choose one Attribute Proficiency. Then distribute each attribute's modifier across its sub-Practices (see [Practice/skill System Overview](https://app.notion.com/p/38eed8479ccd803b9544f1d0ce3d97cf) for full rules — Physical +5 might become +2 Prowess, +4 Finesse, -1 Fieldcraft; you cannot go lower than -1)

 redistribution allowed at L4/8/12/16/20 with DM approval, full Respec of capabilities and primitives too if dm allows, as long as it makes sense)

When making checks using your proficient attribute, add your **Proficiency Bonus (PB)**.

**Starting Proficiency Bonus:** +2

Your **DC** is 5 + attribute proficient + PB. 

Everyone has a base of **30ft** of movement. 

### Example

Proficient Attribute: Physical

- Physical +5 (+PB separately bc proficient)
- Mental +3
- Magical +2

---

# ⚙️ STEP 4 — BU INITIALIZATION

Player begins with **25 BU** at lvl 1

The player may distribute this freely between:

- Race
- Background
- Build / Archetype

Recommended budgets:

- Race: 6–10 BU
- Background: 4–6 BU
- Build: remainder

These recommendations are not mandatory.

Player receives BU pool based on level 1 start.

BU is used to:

- unlock grammar blocks
- buy proficiencies
- upgrade vitality
- enable utilities

Library heritage auto-spend BU internally if selected.

---

# 🧩 STEP 5 — GRAMMAR BUILD

Player unlocks capability grammar (primitives):

- Verb access (what actions exist)
- Domain access (what forces they use)
- Axis control (range / area / duration / targets)
- Composition slots (how complex abilities can be)

This defines what the character CAN express.

---

# 🔥 STEP 6 — CAPABILITY CREATION

Player constructs abilities using grammar:

Example:

- Fire + DAMAGE + AoE → Fireburst
- Gravity + CONTROL + AREA → Gravity field

Capabilities may also come from:

- Library heritage
- Custom construction

---

# 🧠 STEP 7 — PROFICIENCIES & UTILITIES

BU can also be spent on:

- skill/practice proficiencies
- expertise upgrades
- tools & languages
- reactions
- passive abilities

---

# ❤️ STEP 8 — VITALITY FINALIZATION

Compute base vitality:

- system formula Vitality = (10 + Proficiency Bonus) × Level
- BU modifiers
- capability modifiers

Defines survival + resource economy.

---

# 🎯 STEP 9 — DM VALIDATION

DM checks:

- grammar consistency
- Tier emergence estimation
- BU balance sanity
- narrative coherence

No hard rejection unless broken.

---

# 🧠 SYSTEM SUMMARY

Character creation =

Concept → DBP Frame → BU Pool → Grammar → Capabilities → Skills → Vitality → DM Pass

---

# ⚖️ DESIGN PRINCIPLE

Players do NOT pick abilities.

They construct actions from grammar rules.

---

# 🔥 RESULT

✔ fast onboarding

✔ deep custom builds

✔ library compatibility

✔ full improvisation support

## 📖 Glossary

- **BU** = Build Unit. The universal construction currency used to purchase primitives globally on a character's ledger. One-time purchases that unlock tier licenses forever.
- **DBP** = Distributed Build Points. A separate optional pool (when used by your table) for distributing character options like race traits, background features, or starting equipment that don't fit cleanly into the BU system. DBP is a flavor allocation system, not a primitive-purchase mechanism.

The character creation flow uses **DBP for the Base Frame** (Step 2), then transitions to **BU for capability construction** (Step 4+).
